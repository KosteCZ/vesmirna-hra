Mam napad. Chci vytvorit hru pro vice hracu, kterou by mohli hrat kamaradi pres internet. Hra bude vesmes tabulkova. Tematicky by hra byla z prostredi vesmirne planety, na ktere by hraci jakoze pristali. Hraci by pro zacatek rekneme jen tezili par surovin - zvysovali by se jim cisla v tabulce v pruhebu casu. Mohli by vylepsovat tezici budovi a az by vytezili dostatek surovin a vylepsili budovy, tak by vyhrali. Potrebuji poradit s vyberem technologii, aby bylo mozne hru hrat pres internet nebo na internetu a aby to bylo pokud mozno zadarmo.

Odpověď Gemini
To je skvělý nápad! Klasické tabulkové budovatelské nebo "idle" hry (ve stylu legend jako OGame, Travian nebo Cookie Clicker) jsou fantastickým projektem pro začátek. Jsou zábavné na hraní s kamarády a dají se naprogramovat relativně snadno, protože nepotřebuješ řešit složitou 3D grafiku nebo fyzikální enginy.

Aby to bylo multiplayerové, přístupné přes prohlížeč a ideálně zcela zdarma, postavíme to na moderních webových technologiích. K tomu, aby hra fungovala, potřebuješ tři hlavní části: Frontend (to, co vidí hráč), Backend (mozek hry) a Databázi (kde jsou uložené stavy účtů a suroviny).

Zde je doporučený technologický stack (výběr technologií), který je pro tvůj účel naprosto ideální a má štědré plány zdarma (Free Tiers):

1. Frontend (Uživatelské rozhraní)
Protože půjde o tabulky, čísla a tlačítka, nepotřebuješ herní enginy jako Unity. Úplně si vystačíš se standardními nástroji pro tvorbu webu.

Technologie: HTML, CSS a JavaScript. Pro snadnější práci s tabulkami a plynulou aktualizaci čísel (aby se stránka nemusela pořád obnovovat) doporučuji použít jednoduchý framework, jako je Vue.js, Svelte nebo React. Svelte je obzvlášť skvělé pro začátečníky, protože jeho kód vypadá skoro jako čisté HTML a JavaScript.

Hosting zdarma: Svoji webovou stránku můžeš zcela zdarma nahrát (hostovat) na služby jako Vercel, Netlify nebo GitHub Pages. Tyto služby jsou stavěné přímo na frontendové aplikace a jsou bleskově rychlé.

2. Backend a Databáze (Dvě cesty)
Tady se láme chleba. Potřebuješ, aby spolu hráči mohli komunikovat (nebo se alespoň vidět v žebříčku) a aby jejich postup zůstal uložený.

Cesta A: Backend jako Služba (BaaS) – 🏆 Nejlepší volba pro tebe
Místo toho, abys programoval vlastní server od nuly, použiješ hotovou službu, která za tebe řeší databázi, přihlašování uživatelů a real-time (okamžitou) synchronizaci.

Supabase nebo Firebase.

Proč to zvolit: Obě služby mají obrovský tarif zdarma, který pro hru s kamarády nikdy nevyčerpáš. Nabízejí tzv. "real-time databázi". Pokud hráč A vylepší budovu, do databáze se to zapíše a databáze sama pošle signál hráči B, kterému se okamžitě aktualizuje tabulka na obrazovce. Zvládnou i přihlašování (např. přes e-mail nebo Google účet).

Cesta B: Vlastní server v Node.js – 🛠️ Více kontroly, více práce
Pokud chceš mít absolutní kontrolu nad tím, jak hra počítá suroviny na pozadí a chceš se naučit psát vlastní backend.

Technologie: Node.js + framework Express + knihovna Socket.io (pro okamžitou komunikaci mezi hráči).

Databáze: MongoDB Atlas (mají cloudovou databázi zdarma navždy).

Hosting zdarma: Render.com nebo Railway.app. Mají tarify zdarma, ale u Renderu server po 15 minutách nečinnosti "usne" a první probuzení pak trvá třeba 30 sekund.

Krok 1: Příprava databáze (Supabase)
Než začneme programovat, musíme říct Supabase, jaká data budeme ukládat. Přejdi ve svém Supabase projektu do záložky Table Editor a vytvoř novou tabulku. Nazveme ji třeba planets (každý hráč bude mít jednu planetu).

Vytvoř v ní následující sloupce (Columns):

id (typ: uuid, Primary Key) – unikátní identifikátor planety/hráče.

player_name (typ: text) – jméno hráče.

iron_amount (typ: numeric nebo float4) – kolik má hráč aktuálně vytěženého železa. Výchozí hodnota (Default): 0.

mine_level (typ: int4) – úroveň těžební budovy. Výchozí hodnota: 1.

last_updated (typ: timestamptz) – přesný čas, kdy se naposledy stav surovin uložil do databáze. Výchozí hodnota: now().

Poznámka: Pro jednoduchost zatím ignorujeme složité přihlašování (Auth) a budeme brát co řádek, to jeden hráč.

To je ta nejzábavnější část! Aby hra působila živě, musíme ve Svelte vytvořit tzv. herní smyčku (game loop).

Vtip je v tom, že budeme mít dvě hodnoty:

Serverová hodnota: Ta, kterou jsme stáhli ze Supabase (statické číslo).

Klientská hodnota (Live): Tu budeme v prohlížeči každých 100 milisekund navyšovat, aby hráč viděl, jak mu suroviny rostou.

Tady je upravený kód pro tvůj soubor src/App.svelte. Přidal jsem tam logiku, která počítá přírůstek na základě úrovně dolu:

Svelte
<script>
  import { onMount, onDestroy } from 'svelte';
  import { supabase } from './supabase.js';

  let planet = null;
  let loading = true;
  let displayIron = 0; // Tohle číslo uvidí hráč "tikat"
  let interval;

  // TVOJE ID ZE SUPABASE (vlož sem to svoje z tabulky planets)
  const TEST_PLANET_ID = 'tvoje-uuid-z-databaze'; 

  // Konstanta: Kolik železa vyprodukuje 1. úroveň dolu za sekundu
  const BASE_PRODUCTION = 1; 

  onMount(async () => {
    // 1. Načtení dat ze Supabase
    const { data, error } = await supabase
      .from('planets')
      .select('*')
      .eq('id', TEST_PLANET_ID)
      .single();

    if (error) {
      console.error("Chyba:", error);
    } else {
      planet = data;
      
      // 2. Výpočet reálného stavu (pokud byl hráč dlouho pryč)
      const now = new Date();
      const lastUpdate = new Date(planet.last_updated);
      const secondsElapsed = (now - lastUpdate) / 1000;
      const productionPerSec = planet.mine_level * BASE_PRODUCTION;
      
      // Přičteme to, co se vytěžilo offline
      displayIron = planet.iron_amount + (secondsElapsed * productionPerSec);
      
      // 3. Spuštění vizuálního "tikání" (každých 100ms)
      interval = setInterval(() => {
        // Přičteme desetinu vteřinové produkce
        displayIron += (productionPerSec / 10);
      }, 100);
    }
    loading = false;
  });

  // Funkce pro vylepšení dolu
  async function upgradeMine() {
    const cost = 100 * planet.mine_level; // Cena roste s úrovní
    
    if (displayIron >= cost) {
      const newLevel = planet.mine_level + 1;
      const newIron = displayIron - cost;

      // Uložíme do Supabase
      const { data, error } = await supabase
        .from('planets')
        .update({ 
          mine_level: newLevel, 
          iron_amount: newIron,
          last_updated: new Date().toISOString() 
        })
        .eq('id', planet.id)
        .select()
        .single();

      if (!error) {
        planet = data;
        displayIron = newIron;
        alert("Důl vylepšen!");
      }
    } else {
      alert("Nemáš dost železa!");
    }
  }

  // Uklidíme interval při zavření stránky
  onDestroy(() => {
    if (interval) clearInterval(interval);
  });
</script>

<main>
  <h1>🚀 Vesmírná Kolonie</h1>

  {#if loading}
    <div class="loader">Přistávám na planetě...</div>
  {:else if planet}
    <div class="card">
      <h2>Velitel: {planet.player_name}</h2>
      
      <div class="resource-box">
        <span class="label">Sklad železa:</span>
        <span class="value">{Math.floor(displayIron)}</span> 
        <small>({(planet.mine_level * BASE_PRODUCTION).toFixed(1)}/s)</small>
      </div>

      <div class="building-box">
        <p><strong>Důl na železo (Úroveň {planet.mine_level})</strong></p>
        <button on:click={upgradeMine} disabled={displayIron < 100 * planet.mine_level}>
          Vylepšit za {100 * planet.mine_level} železa
        </button>
      </div>
    </div>
  {:else}
    <p>Chyba při spojení s orbitální stanicí.</p>
  {/if}
</main>

Co se v kódu děje:
Offline těžba: Když hru zapneš, program spočítá rozdíl mezi "teď" a časem posledního uložení (last_updated). Tím hráč dostane suroviny za dobu, co byl pryč.

setInterval: Každých 100 milisekund přičte kousek železa k proměnné displayIron. Díky tomu číslo na obrazovce krásně plynule roste.

Vylepšení (Upgrade): Tlačítko odečte suroviny a pošle do Supabase nový stav (úroveň dolu a zbytek železa).

Důležitý krok v Supabase:
Aby ti fungovalo ukládání (update), musíš v Supabase vypnout (nebo nastavit) tzv. RLS (Row Level Security). Pro začátek a testování s kamarády můžeš v Table Editoru u tabulky planets kliknout na RLS disabled (nebo vytvořit "Policy", která dovolí komukoliv číst i zapisovat – ale pozor, to je jen pro testování!).

2. Kód pro žebříček ve Svelte
Upravíme tvůj soubor App.svelte. Přidáme novou proměnnou allPlanets a funkci, která bude hlídat změny u ostatních hráčů.

Svelte
<script>
  import { onMount, onDestroy } from 'svelte';
  import { supabase } from './supabase.js';

  let planet = null;
  let allPlanets = []; // Tady budou data pro žebříček
  let loading = true;
  let displayIron = 0;
  let interval;

  const TEST_PLANET_ID = 'tvoje-uuid-z-databaze'; 
  const BASE_PRODUCTION = 1;

  onMount(async () => {
    // 1. Načtení tvé planety (stejné jako předtím)
    const { data: myData } = await supabase.from('planets').select('*').eq('id', TEST_PLANET_ID).single();
    if (myData) {
      planet = myData;
      const secondsElapsed = (new Date() - new Date(planet.last_updated)) / 1000;
      displayIron = planet.iron_amount + (secondsElapsed * planet.mine_level * BASE_PRODUCTION);
      
      interval = setInterval(() => {
        displayIron += (planet.mine_level * BASE_PRODUCTION / 10);
      }, 100);
    }

    // 2. Načtení všech planet pro žebříček
    await fetchLeaderboard();

    // 3. REÁLNÝ ČAS: Sledujeme změny v tabulce planets
    const channel = supabase
      .channel('schema-db-changes')
      .on('postgres_changes', { event: '*', schema: 'public', table: 'planets' }, () => {
        fetchLeaderboard(); // Když se cokoliv změní, aktualizuj žebříček
      })
      .subscribe();

    loading = false;
  });

  async function fetchLeaderboard() {
    const { data } = await supabase
      .from('planets')
      .select('player_name, mine_level, iron_amount')
      .order('mine_level', { ascending: false }) // Seřadit podle úrovně dolu
      .limit(10);
    allPlanets = data || [];
  }

  // ... (ponech funkci upgradeMine z minulého kroku) ...
</script>

        <h2>Tvoje Základna: {planet?.player_name}</h2>

Co se změnilo?
fetchLeaderboard: Funkce, která si ze Supabase vytáhne top 10 hráčů seřazených podle toho, kdo má nejvyšší úroveň dolu.

supabase.channel: Tohle je to "kouzlo". Vytvoří to otevřené spojení se serverem. Jakmile tvůj kamarád klikne na "Upgrade", tvůj prohlížeč dostane zprávu a okamžitě zavolá fetchLeaderboard, aby se data na tvé obrazovce změnila, aniž bys musel stránku obnovovat.

HTML Tabulka: Použili jsme Svelte blok {#each ...}, který projde seznam planet a vykreslí řádky tabulky.

################################# UPRAVENA FUNKCI VERZE (spojeni 2 spolednich, aby funovali): #######################################

<script>
  import { onMount, onDestroy } from 'svelte';
  import { supabase } from './supabase.js';

  let planet = null;
  let allPlanets = []; // Tady budou data pro žebříček
  let loading = true;
  let displayIron = 0; // Tohle číslo uvidí hráč "tikat"
  let interval;

  // TVOJE ID ZE SUPABASE (vlož sem to svoje z tabulky planets)
  const TEST_PLANET_ID = '59e06ed1-fbdb-45a0-9038-e8b78e59b026'; 

  // Konstanta: Kolik železa vyprodukuje 1. úroveň dolu za sekundu
  const BASE_PRODUCTION = 1; 

  onMount(async () => {
    // 1. Načtení dat ze Supabase
    const { data, error } = await supabase
      .from('planets')
      .select('*')
      .eq('id', TEST_PLANET_ID)
      .single();

    if (error) {
      console.error("Chyba:", error);
    } else {
      planet = data;
      
      // 2. Výpočet reálného stavu (pokud byl hráč dlouho pryč)
      const now = new Date();
      const lastUpdate = new Date(planet.last_updated);
      const secondsElapsed = (now - lastUpdate) / 1000;
      const productionPerSec = planet.mine_level * BASE_PRODUCTION;
      
      // Přičteme to, co se vytěžilo offline
      displayIron = planet.iron_amount + (secondsElapsed * productionPerSec);
      
      // 3. Spuštění vizuálního "tikání" (každých 100ms)
      interval = setInterval(() => {
        // Přičteme desetinu vteřinové produkce
        displayIron += (productionPerSec / 10);
      }, 100);
    }
	
	// 2. Načtení všech planet pro žebříček
    await fetchLeaderboard();

    // 3. REÁLNÝ ČAS: Sledujeme změny v tabulce planets
    const channel = supabase
      .channel('schema-db-changes')
      .on('postgres_changes', { event: '*', schema: 'public', table: 'planets' }, () => {
        fetchLeaderboard(); // Když se cokoliv změní, aktualizuj žebříček
      })
      .subscribe();

    loading = false;
  });
  
  async function fetchLeaderboard() {
    const { data } = await supabase
      .from('planets')
      .select('player_name, mine_level, iron_amount')
      .order('mine_level', { ascending: false }) // Seřadit podle úrovně dolu
      .limit(10);
    allPlanets = data || [];
  }

  // Funkce pro vylepšení dolu
  async function upgradeMine() {
    const cost = 100 * planet.mine_level; // Cena roste s úrovní
    
    if (displayIron >= cost) {
      const newLevel = planet.mine_level + 1;
      const newIron = displayIron - cost;

      // Uložíme do Supabase
      const { data, error } = await supabase
        .from('planets')
        .update({ 
          mine_level: newLevel, 
          iron_amount: newIron,
          last_updated: new Date().toISOString() 
        })
        .eq('id', planet.id)
        .select()
        .single();

      if (!error) {
        planet = data;
        displayIron = newIron;
        alert("Důl vylepšen!");
      }
    } else {
      alert("Nemáš dost železa!");
    }
  }

  // Uklidíme interval při zavření stránky
  onDestroy(() => {
    if (interval) clearInterval(interval);
  });
</script>

<main>
  <h1>🚀 Vesmírná Kolonie</h1>

  {#if loading}
    <div class="loader">Přistávám na planetě...</div>
  {:else if planet}
    <div class="game-container">
